The OSI Model

The Open Systems Interconnection model is a conceptual model that characterizes and standardizes the internal functions of a communication system by partitioning it into abstraction layers
- Wikipedia


7 Application

Serves as the window for users and application processes to access the network services

The Application layer is where the users actually communicate to the computer, it comes into play only when it's clear that access to the network will be needed soon. For example, if you could remove every aspect of networking components such as TCP/IP and the NIC card, your web browser would still be able to view a local HTML document.

The Application layer works as the interface between actual application programs. End-user programs such as Microsoft Word don't exist at the Application layer itself, they interface with its protocols.


6 Presentation

Formats the data to be presented to the Application layer, it can be viewed as the Translator for the network

This layer presents data to the Application layer and is responsible for data translation and code formatting. It's much like the OSI model's translator, providing coding and version services.

Computers are configured to receive this generically formatted data and then reformat it into its native state. Data compression, decompression, encryption, and decryption are associated with this layer because of this.


5 Session

Allows session establishment between processes running on different stations

The Session layer is responsible for setting up, managing, and taking apart sessions between Presentation layer entities and keeping user data separate. Dialog control between devices also happens here.

Communication between hosts various applications at the Session layer are coordinated and organized via three different modes: simplex, half-duplex, full-duplex.


4 Transport

Ensures that messages are delivered error-free, in sequence, and with no losses or duplications

This layer segments and reassembles data into a single data stream. Services here take all the data received from the upper levels and then combines it into a concise data stream.

These protocols provide end-to-end data transport services and establish logical connections between the sending host and destination on an internetwork. TCP and UDP are integral to this layer.


3 Network

Controls the operation of the subnet, deciding which physical path the data takes

The Network layer manages device addressing, tracks the location of devices on the network, and determines the best way to move data. It's up to this layer to transport traffic between devices that aren't locally attached.

Routers are specified at this layer and provider the routing services within internetworks.


2 Data Link

Provides error-free transfer of data frames from one node to another over the Physical layer

The Data Link layer handles physical transmission of data, error notification, network topology, and flow control. It ensures that data is delivered to the proper device on a LAN, and translates messages from the Network layer into bits for the Physical layer.

Layer 2 switching is considered hardware-based bridging because it uses specialized hardware called an application-specific integrated circuit (ASIC)


1 Physical

Concerned with the transmission and reception of the unstructured raw bit stream over the physical medium

Finally, at the bottom, the Physical layer does two things: it sends and receives bits.
